## 1. الأدوار الخمسة

| الدور | الاسم بالعربية | المسؤولية الرئيسية |
|---|---|---|
| `owner` | المالك / الأدمن العام | صلاحية كاملة على كل شيء (Master) |
| `product_manager` | مدير المنتجات | إضافة وتعديل المنتجات، الفئات، المخزون |
| `order_manager` | مدير الطلبات | متابعة الطلبات، تحديث الحالات، تعيين المندوبين |
| `finance` | المالية | التحقق من التحويلات، المصاريف، الرواتب، التقارير |
| `delivery` | مندوب توصيل | يرى طلباته فقط ويُحدّث حالة التوصيل |

> الأدوار محفوظة في `users.role` (مع `account_type='staff'`) + جدول `team_members` للمعلومات الإضافية (الراتب، الحالة، تاريخ الانضمام).

---

## 2. كيف يدخل الموظف؟

| الخطوة | التفاصيل |
|---|---|
| **1. الإنشاء** | الأدمن يفتح `/admin` → تبويب "الفريق" (`TeamManagement`) → "إضافة موظف" → يُدخل الاسم/الإيميل/كلمة المرور/الدور. |
| **2. المشاركة** | الأدمن يرسل للموظف: رابط `[domain]/staff` + إيميله + كلمة المرور. |
| **3. الدخول** | الموظف يفتح `/staff` → يُدخل بياناته → يدخل تلقائياً لـ `StaffPortal` المخصّص لدوره. |
| **4. التعطيل** | الأدمن يُغيّر دور الموظف إلى `customer` — يبقى في DB لكن يفقد الوصول. |

> **ملاحظة أمنية:** كلمات المرور مُشفّرة بـ bcrypt. لا تشاركها عبر واتساب — استخدم رسالة آمنة.

## 3. صلاحيات كل دور بالتفصيل

### 👑 المالك (`owner`)
✅ كل الصلاحيات أدناه + إدارة الفريق + `/admin/ai-agents` + الإعدادات العامة + التسعير + البانرات.
🔑 يصل أيضاً لـ `/admin` بـ admin token المنفصل.

### 📦 مدير المنتجات (`product_manager`)
| يستطيع | لا يستطيع |
|---|---|
| `GET/POST/PUT /api/staff/products` | حذف منتج (مخصّص للأدمن) |
| إضافة/تعديل الفئات | الوصول لبيانات المالية |
| تحديث المخزون | تعديل الأسعار العامة لو محظور في الإعدادات |
| رفع صور المنتجات | إدارة الموظفين |

**الصفحات:** تبويب "المنتجات" + "المخزون" في `StaffPortal`.

### 🛒 مدير الطلبات (`order_manager`)
| يستطيع | لا يستطيع |
|---|---|
| `GET /api/staff/orders` (كل الطلبات) | حذف طلب |
| `PUT /api/staff/orders/:id/status` | تعديل المنتجات أو الأسعار |
| `PUT /api/staff/orders/:id/assign` (تعيين مندوب) | الوصول للمالية |
| التحقق من إيصالات الدفع | إنشاء مستخدمين |
| الاتصال/واتساب العملاء من اللوحة | |

**الصفحات:** تبويب "الطلبات" — قائمة شاملة + فلاتر حالة.

### 💰 المالية (`finance`)
| يستطيع | لا يستطيع |
|---|---|
| `GET /api/staff/financial-summary` | تعديل المنتجات |
| التحقق من التحويلات البنكية | إدارة الفريق |
| `GET/POST /api/finance/expenses` (مصاريف) | تغيير حالة الطلب يدوياً |
| `GET /api/finance/payroll` (رواتب) | |
| إدارة الأصول | |
| توليد تقارير الأرباح | |

**الصفحات:** تبويب "المالية" — لوحة بمؤشرات الإيرادات/المصاريف/الأرباح + سجل الرواتب.

### 🚚 مندوب التوصيل (`delivery`)
| يستطيع | لا يستطيع |
|---|---|
| `GET /api/staff/orders?assigned_to_me=true` فقط | رؤية طلبات غيره |
| `PUT /api/staff/orders/:id/status` لطلباته فقط (إلى `shipped`/`delivered`/`failed`) | تعديل المنتجات أو الأسعار |
| تأكيد استلام النقد عند التسليم | الوصول للمالية |
| تسجيل الحضور `POST /api/staff/attendance/checkin` | |

**الصفحات:** تبويب "طلباتي" — قائمة مفلترة + زر "تم التوصيل" لكل طلب.

## 4. الحماية على مستوى الخادم

كل المسارات `/api/staff/*` محمية بـ **middleware موحّد**:

```ts
requireStaff(allowedRoles: string[])
```

- يتحقق من جلسة المستخدم.
- يتحقق أن `user.account_type === 'staff'` وأن `user.role` ضمن `allowedRoles`.
- يُرجع `403 ليس لديك صلاحية` عند الفشل.

**مثال:**
```ts
app.put('/api/staff/orders/:id/status',
  requireStaff(['owner', 'order_manager', 'delivery']),
  handler);
```

> المندوب يصل لنفس المسار لكن `handler` يتحقق إضافياً أن `order.assigned_to === user.id` قبل السماح.

## 5. سجل العمليات (Audit)

كل تغيير حساس يُسجَّل في `audit_log`:
- تغيير حالة الطلب → `who/what/when/old/new`.
- إضافة مصروف → `staff_id`, `amount`, `category`.
- تعيين مندوب → `order_id`, `from_user`, `to_user`.

يستطيع المالك مراجعة السجل من `/admin` → تبويب "السجل".

---

## 6. أفضل الممارسات للأدمن

1. **مبدأ أقل صلاحية:** أعطِ الموظف الدور الأدنى الذي يكفيه. لا تجعل الجميع `owner`.
2. **كلمات مرور قوية:** 12 حرفاً+ مع رموز. غيّرها إذا غادر الموظف.
3. **مراجعة شهرية:** افتح "الفريق" وتأكد أن كل النشطين فعلاً يعملون. أوقف الباقين.
4. **المندوبون:** أنشئ حساباً لكل مندوب — لا تشارك حساباً واحداً بين عدة مندوبين (لأن `assigned_to` يصبح مبهماً).
5. **المالية حسّاسة:** أعطِ دور `finance` لشخص واحد فقط أو اثنين بحدّ أقصى.

## 7. خارطة طريق مستقبلية (غير منفّذة)

- 🟡 **2FA:** رمز SMS عند تسجيل دخول الموظف الحسّاس.
- 🟡 **صلاحيات دقيقة:** خانات `permissions` JSONB في `team_members` تسمح بتخصيص داخل نفس الدور.
- 🟡 **حدّ زمني:** قفل جلسة الموظف بعد عدم نشاط 30 دقيقة.
- 🟡 **IP whitelist:** المالية والأدمن من IP محدد فقط.
