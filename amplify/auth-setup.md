## خطوات تهيئة المصادقة عبر AWS Amplify (قابلة للنسخ)

افتح الطرفية (Terminal) في بيئتك (Replit أو محليًا) ونفّذ الأوامر التالية بالترتيب:

1) تهيئة ملفات Amplify (يقوم بإنشاء مجلدات وإعدادات محلية):

```bash
npm create amplify@latest -y
```

قد يطلب منك بعض الخيارات (region، طريقة المصادقة)، أجب بحسب احتياجات المشروع.

2) بعد اكتمال التهيئة، أضف الملفات وادفعها إلى GitHub:

```bash
git add .
git commit -m "إضافة نظام المصادقة لـ أويو بلاست"
git push origin main
```

ملاحظة:
- تأكد أن الفرع الذي تستخدمه هو "main"، وإذا كان اسمه مختلفًا غيّر الأمر الأخير إلى اسم الفرع المناسب.
- لا تقم بدفع ملفات التكوين التي تحتوي مفاتيح (مثل aws-exports.js) إلى المستودع إذا كانت تتضمن بيانات حساسة؛ بدلاً من ذلك أضفها إلى .gitignore أو استخدم Secrets/Environment variables في منصة النشر.

بعد الرفع، إذا كان المشروع مرتبطًا بـ AWS Amplify Console، سيبدأ البناء والنشر تلقائيًا ويظهر مفعول المصادقة في التطبيق بحسب الإعدادات التي اخترتها.
